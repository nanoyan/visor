Status Map

Type: object

Data structure to specify recoding based on response state.

root status
Type: enum (of string)

Status of a response to be recoded.

Must be one of:

  • "UNSET"
  • "NOT_REACHED"
  • "DISPLAYED"
  • "INVALID"
  • "DERIVE_ERROR"
  • "NO_CODING"
  • "CODING_INCOMPLETE"
  • "CODING_ERROR"

root recodeToMissing
Type: string

Identifier for missing. A missing list is required to lookup the code value.

Must match regular expression: ^[A-Za-z_]+$
Examples:

"mbi"

"mbd"

"mir"

root recodeToFalse
Type: boolean

If true, the value is recoded to the first score 0 code found in the code list of coding scheme.
